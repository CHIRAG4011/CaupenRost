Common Fixes
  • Check that MONGO_URI is set correctly in your environment secrets — copy it exactly from MongoDB Atlas (no trailing spaces).
  • In MongoDB Atlas, go to Network Access and add 0.0.0.0/0 to allow connections from any IP (required for serverless/Vercel deployments).
  • Confirm your Atlas database user credentials — the username and password in the URI must match the database user, not your Atlas login.
  • Make sure the database name at the end of your URI (e.g. /caupenrost) exists and the user has read/write access to it.
  • The connection timeout is 5 seconds. If your Atlas cluster is on the free tier and has been idle, it may need a moment to resume — try refreshing once more.
